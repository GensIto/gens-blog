どうやら Sqlite にはトランザクション機能がないみたい?となったので調べてみた
[microsoft の docs](https://learn.microsoft.com/ja-jp/dotnet/standard/data/sqlite/transactions)を確認すると

> SQLite では、データベース内で一度に 1 つのトランザクションに変更を保留することが許可されます。

とあるのでどうやら Sqlite にはトランザクション機能はあるみたいだが、[D1](https://github.com/cloudflare/workers-sdk/issues/2733)にはないようだ。
[公式ドキュメント](https://developers.cloudflare.com/d1/worker-api/d1-database/)を確認すると

> Batched statements are SQL transactions. If a statement in the sequence fails, then an error is returned for that specific statement, and it aborts or rolls back the entire sequence.

とあるので、batch 機能で代替えできるみたい
参考記事:

- [Cloudflare d1 + Prisma をやめて Drizzle orm に移行した](https://noh.ink/articles/Ttbwg2gbzc9Sczn0kzJG)
- [Cloudflare D1 でトランザクションが使えない問題](https://zenn.dev/minamiso/articles/885adef7098ebe)

# batch 機能を使ってトランザクションを実装する

batch 機能を使ってトランザクションを実装する。

```typescript
async createWorkspace(workspace: Workspace): Promise<Workspace> {
    const membership = WorkspaceMembership.createOwnerMembership(
      workspace.workspaceId,
      workspace.ownerUserId
    );

    const results = await this.db.batch([
      this.workspaceRepository.createBuilder(workspace),
      this.workspaceMemberships.createBuilder(membership),
    ]);

    const createdWorkspaceResult = results[0];
    if (!createdWorkspaceResult || createdWorkspaceResult.length === 0) {
      throw new Error("Failed to create workspace");
    }

    const createdWorkspace = createdWorkspaceResult[0];
    return Workspace.of(
      WorkspaceId.of(createdWorkspace.workspaceId),
      WorkspaceName.of(createdWorkspace.name),
      UserId.of(createdWorkspace.ownerUserId)
    );
  }
```

batch には、`createBuilder`メソッドを使ってビルダーを作成する。
なんというか純粋な SQL じゃないとダメみたい

```typescript
createBuilder(workspace: Workspace) {
  return this.db.insert(workspaces).values(workspace.toJson()).returning();
}
```
